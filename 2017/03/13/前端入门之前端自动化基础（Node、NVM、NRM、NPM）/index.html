前端入门之前端自动化基础（Node、NVM、NRM、NPM）

发表于 2017-03-13 | 分类于 前端自动化

1、NodeJS概述

NodeJS是Server side Javascript runtime, 服务端的一个JS运行时，就是一个JS运行环境。

2、NVM，Node版本控制管理工具

NVM（Node Version Manager）Node版本管理工具，为什么需要版本管理呢？

  • node版本更新快
  • 不同的项目依赖不同的node版本，项目之间经常要切换Node版本

2.1环境变量;

  • 环境变量的变量名是不区分大小写的
  • PATH变量是一个特殊值，变量名只能是path/PATH,追加在该路径后面的所以程序，都可以再全局目录下，直接运行相关exe程序。
  • 环境变量包括：用户环境变量与系统变量。用户环境变量就是window当前用户的环境变量，其他用户不能访问。系统变量指的是所用当前系统用户共享的变量；建议将自己配置的环境变量放在用户变量中，用户变量比较干净

2.2如何设置环境变量（win10）

此电脑——>右击属性-高级系统设置——>高级——>环境变量——>用户变量——>新建变量——>在path变量中追加刚才新建的变量。


2.3window版本NVM的安装

2.3.1第一步：

github.com上下载nvm安装包，nvm-noinstall.zip;

2.3.2第二步：

解压nvm-noinstall.zip,然后把文件夹重命名为nvm。在自己喜欢的盘符上（我这里放在C:\根目录下）新建一个文件夹。比如：FrontEndDev，最后把刚刚nvm文件夹复制到FrontEndDev文件夹下。

2.3.3第三步：

以管理员身份双击运行FrontEndDev里面的nvm下的install.cmd文件,在FrontEndDev目录下新建一个setting.txt文本文件，以下是输入新建的txt文件要保存到哪个路径下（必须是nvm.exe同级目录）提示如下：

2.3.4第四步：

创建成功后会自动弹出setting.txt，根据实际情况需要配置选项如下：

root:表示nvm所在的根目录；
path：表示nodejs所在的目录，这个目录会自动生成，它与nvm目录平级；
arch：表示当前64位操作系统；
proxy：表示使用代理下载；
node_mirror：使用国内淘宝node镜像安装node；
npm_mirror：使用国内淘宝npm镜像安装npm；

2.3.5第五步：

配置node、nvm、npm环境变量;

最后向用户变量中添加环境变量：PATH变量中追加%NVM_HOME%;%NVM_SYMLINK%;%NPM_HOME%；

2.3.6第六步：

验证是否安装成功

2.3.7第七步：

安装nodejs/NPM的版本
如果你的电脑系统是32 位的，那么在下载nodejs版本的时候，一定要指明 32 如：nvm install 5.11.0 32这样在32位的电脑系统中，才可以使用，默认是64位的。
