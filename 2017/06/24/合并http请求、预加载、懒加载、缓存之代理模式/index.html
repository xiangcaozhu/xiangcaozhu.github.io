代理模式理解

我们先思考一个问题：为什么每个明星都会有经纪人？又或者为什么房地产中介的销售都自称经纪人？
我们在百度百科查看一下“经纪人”是指：

在经济活动中，以收取佣金为目的，为促成他人交易而从事居间，行纪或者代理等经纪业务的公民，法人和其他经济组织。

经纪人现实生活中的作用

在发展经济，促进流通，繁荣市场等方面发挥着明显的积极作用。
信息来源的多样性，使经纪人在解决商品供求矛盾具有独特的功能。
（1）社会关系多，市场信息灵，办事效率高，帮助企业解决供求矛盾，打开市场销路。
（2）从中居间协调，提供信息，介绍客户，撮合成交。

代理模式在软件设计中的作用

代理模式条件：

本体（房东或明星）很忙，每天有很多人找本体，因此需要代理人从中筛选，再交给本体处理。在开发中，经常遇到的问题：服务器资源无法支撑大量客户端频繁发起的http请求数的矛盾。

代理模式作用：

  • 代理对象可以拥有更多本体不具备的能力
  • 代理对象熟悉本体的特性，能帮本体做出最优的处理。
  • 分离多项职责，实现单一职责原则（只做一件事）
  • 使服务器资源分配更有效率
  • 减轻服务器访问的压力

代理模式概念

当客户不方便直接访问一个对象或者不满足需求的时候，提供一个替身对象来控制这个对象的访问，客户实际上访问的是替身对象。替身对象对请求做出了一些处理后，再把请求转交给本体对象。

代理与本体关系

  • 代理必须与本体接口保存一致
  • 在任何使用本体的地方都可以替换成使用代理。
  • 用户可以放心地使用代理，都可以得到想要的结果。

代理模式的结构

代理具有本体相同的方法，不同本体的是，代理会把传过来的对象，优化后，再转交给本体方法去执行。

保护代理

过滤本体不想处理的事，控制不同权限的对象对目标对象的访问，由于我们无法判断谁访问了某个对象，因此JavaScript不容易实现。

虚拟代理

把一些开销很大的对象，延迟到真正需要它的时候在去创建

预加载

  • 本体对象

  • 代理对象

合并http请求

如果不是对实时性要求非常高的系统，可以采用收集一段时间之内的请求，一次性发送给服务器。

惰性加载

先让代理对象给用户提前使用，代理接口与本体接口一致。当用户特定事件触发时，才动态的创建script，往页面中插入js引用。加载完成后，本体对象会遍历一下，刚刚代理对象缓存的函数队列，依依执行一遍。

缓存代理

缓存一些开销大的运算结果提供暂时的储存，下次如果传递的参数跟之前的一致，则直接返回前面的存储结果。
