Flex弹性布局从语法到布局梳理总结

发表于 2017-05-20 | 分类于 页面布局

1、flex布局概述

Flex是Flexible Box的缩写，意为”弹性布局”，用来为盒状模型提供最大的灵活性。

Flex布局，可以简便、完整、响应式地实现各种页面布局。

布局的传统解决方案，基于盒状模型，依赖 display属性 + position属性 + float属性。

任何一个容器都可以指定为Flex布局。

  • 块级元素：display：flex;
  • 内联元素：display：inline-flex;

声明display：flex布局的元素，称为Flex容器（flex container），简称”容器”。
它的所有包含的子元素称为Flex项目（flex item），简称”项目”。
树立器坐标轴概念，即垂直方向的y轴，称为“主轴”，水平方向的x轴，称之为“侧轴，主轴与侧轴相交的格子，称之为“交叉区域”。

设为Flex布局以后，子元素的float、clear和vertical-align属性将失效。

3、flex布局对齐属性的辨析（难点）

3.1单行或单列布局

该布局特点：justify-content或align-conten属性无效，不起作用。只能通过align-item属性来调整，上中下，左中右对齐。

3.2多行或多列布局

该布局特点：当同时设置justify-content与align-content属性时，align-item属性失效，不起作用。当同时设置justify-content与align-content的属性为stretch时，设置align-item属性有效，align-item可以理解为相对于在交叉区域里的上中下，左中右的对齐。（如图1）

3.3flex-item子元素如何分配flex-container空间（难点）

  • 扩展分配flex-grow
    属性定义项目的放大比例，默认为0，即如果存在剩余空间，也不放大。
    如果所有项目的flex-grow属性都为1，则它们将等分剩余空间（如果有的话）。
    如果一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。
  • 收缩分配flex-shrink
    属性定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。
    如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。
    如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。
  • 默认大小flex-basis/width
    属性定义了在分配多余空间之前，项目占据的主轴空间（main size）。
    浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto，即项目的本来大小。它可以设为跟width或height属性一样的值（比如350px），则项目将占据固定空间。

3.3.1计算问题：

  • 剩余空间：flex-container容器空间-flex-items子项总空间=剩余的空间；
  • 扩展系数：flex-items子项里flex-grow的总和，即为扩展比例系数。
  • 收缩系数：flex-items子项里flex-shrink的总和，即为收缩比例系数。

flex-items子项空间：设置了flex-basis或width值。当flex-items子项没有设置flex-basis或width属性值，默认子项空间大小为0，flex-grow或flex-shrink初始缩放大小直接就是相对于flex-container容器大小的计算；当子项内部有内容，内容大小作为子项的大小。
扩展大小：是分配剩余空间 扩展系数；
收缩大小：是分配剩余空间 收缩系数；

5、flex布局实战详细flex布局翻译

5.1网格布局

5.1.1平均分布

  • 适用：框架布局
  • 难点：凹槽，距离真正网格布局还有一定的问题。

5.2圣杯布局

圣杯布局（Holy Grail Layout）指的是一种最常见的网站布局。页面从上到下，分成三个部分：头部（header），躯干（body），尾部（footer）。其中躯干又水平分成三栏，从左到右为：导航、主栏、副栏。

  • 知识点：
    1、一个容器既可以display：flex，也可以设置flex：1；
    2、vh单位设置满屏，min-height: 100vh；而min-height: 100%不可以。
    3、可以根据屏幕大小做响应式网站

5.3流式布局

每行的项目数固定，会自动分行。

  • 难点：1、如何保证平均分配的box始终是正方形。2、box里的元素如何做到垂直居中对齐。
  • 解决：
    1、flex-container设置固定宽度width，确定flex-items个数，并给flex-items设置flex：0 0 100%/5;height:width/5;
    2、在flex-items子项上设置：display: flex;align-items: center;justify-content: center;为了防止子项内容破坏布局，还必须加上overflow:hiddle;。注意的是子项的子项垂直居中对齐是文本节点还是元素节点span无关。

5.4悬挂式布局

有时，主栏的左侧或右侧，需要添加一个图片栏。

  • 难点：order与align-self属性应用

5.5输入框的布局

我们常常需要在输入框的前方添加提示，后方添加按钮。

  • 难点：1、盒模型的两种模式的区别，采用box-sizing设置。2、清楚表达元素的默认样式；3、元素内容的图标文字的垂直居中布局

5.6Sticky Footer固定的底栏布局

有时，页面内容太少，无法占满一屏的高度，底栏就会抬高到页面的中间。这时可以采用Flex布局，让底栏总是出现在页面的底部。当页面内容超过一屏幕高度时，底部自动布局到最后位置上。

  • 理解：flex本身具有sticky footer的特性，具有很好的弹性。可以思考一下传统绝对定位布局方式如何实现改布局？
  • 区别：与圣杯布局对比，发现基本没有太大的区别，仅仅只是布局格局不一样罢了！圣杯布局典型特征就是上下，中间左中右布局。使用flex布局技术原理一样。
