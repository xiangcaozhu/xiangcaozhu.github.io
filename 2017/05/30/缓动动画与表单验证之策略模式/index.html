缓动动画与表单验证之策略模式

发表于 2017-05-30 | 分类于 JavaScript

1、策略模式

条条大路通罗马，每条路就是一种策略，在程序设计中，实现一个功能有很多种方法，可以随意替换。
定义：定义一系列的算法，把他们一个个封装起来，并且使他们可以相互替换。将不变的部分和变化的部分分隔开是每个设计模式的主题，策略模式的目的就是将算法的使用与算法的实现分离开来。
策略类：封装了具体的算法，并负责具体计算过程。广义上也指一系列指向目标一致，可被替换的“业务规则”。
环境类：context，接受客户请求，随后把请求委托给某个策略类。

3、缓动动画

动画的本质：让对应的html元素，按照一定的缓动算法进行运动而已。

基本原理：让差距不大的原画，以较快的帧数播放，达到视觉效果。在JavaScript里面就是通过改变css属性来实现（left、top、background-position、transform等）。

基本思路：

  • 缓动算法四个参数（t: current time（当前时间new Date）；b: beginning value（初始位置startPos）；c: change in value（变化量endPos-startPos）；d: duration（持续时间duration））
  • Animate构造函数、start、step、update三个方法

5、策略模式优缺点

5.1优点

  • 避免多重条件语句
  • 遵循开发-封闭原则
  • 可复用
  • 组合和委托替代实现了继承

5.2缺点：

  • 增加了策略类和策略对象
  • 违反最少知识原则

5.3函数与策略模式

策略模式是隐形的，策略对象可以直接用函数表示，通过把策略对象作为参数在函数中传递，达到多态的效果。
