深入学习面向对象中this、call和apply应用

发表于 2017-05-28 | 分类于 JavaScript

1、this关键字

JavaScript中的this总是指向一个对象，由运行时基于函数执行环境动态绑定，而非函数声明的环境。

1.1this指向

  • 作为对象的方法调用obj.method()，this指向该对象。

  • 作为普通函数调用method(),this指向全局window对象。

  • 作为构造器调用function Person(){this.name},this指向调用该方法的对象。
    当有new运算符调用函数时，该函数始终会返回一个对象，并且构造器里面的this就是指向返回的这个对象。
    当函数内指定return {}字面量对象时，返回只是{}对象，而非原来this对象。当return 值类型时，只是表示函数有返回值，this指向不变。

  • call或apply调用，动态改变this指向
    冒用其他对象或其他构造函数的属性，obj.method.call(obj2,arg1,arg2)或obj.method.apply(obj2,[arg1,arg2])。

    哪个对象调用了该函数，函数里面的this就指向谁。

2、丢失this，普通函数调用

把一个对象obj里的method方法赋值给一个新的变量newMethod时，由于对象赋值是地址引用，newMethod也指向了同一个obj.method里的函数。

这个时候，newMethod(),调用就是普通函数调用了。

3、call或apply

熟练使用这两个方法，是成为正真JavaScript程序员的重要一步。
当我们使用call或apply时，如果我们传入的第一个参数null,函数体内的this会指向默认的宿主对象，在浏览器中是window
