深入学习JavaScript面向对象封装、继承、多态

发表于 2017-05-25 | 分类于 JavaScript

学习建议

在阅读JavaScript原理代码时，可以通过浏览器调试工具去深入学习：1、变量或函数提升；2、js运行机制；3、监听变量变化。很多看不懂，或被忽视的东西都会更好理解。本文是对《JavaScript设计模式与开放实践》这本书的知识梳理。

概述

JavaScript属于动态类型的语言，无需强制数据类型。所谓动态，即运行时编译，优点：代码简洁，专注于业务逻辑。缺点：无法保证变量类型。建立于“鸭子类型”的概念之上，我们可以尝试调用任何对象的任何方法，而无需考虑它原本是否被设计为拥有该方法。什么是鸭子类型呢？通俗讲，不管他是不是鸭子，只要具有鸭子的行为特征，就算是鸭子。即只关注对象的行为，而不是对象本身。

鸭子类型意义，实现面向对象设计一个原则：“面向接口编程，而不是面向实现编程”

多态（polymorphism [,pɒlɪ’mɔːfɪz(ə)m]）

含义：

不同对象调用同一方法或函数，执行的结果不同。

思想：

做什么（普通函数）与谁去做（构造函数）、怎么去（原型对象上的方法）做分离开来。不变的事物与可变的事物区分开来。消除类型之间的耦合关系。但是在JavaScript中，没有类型检查，就造就了JavaScript对象多态性与生俱来。

方法二是使用面向对象继承（实现继承）实现的，在不修改原代码的前提下，达到动态拓展的目的，是优化多态常用手段。

作用：

  • 多态的最根本好处在于，你不必再向对象询问“你是什么类型”，而后根据得到的答案去调用对象的某个行为。
  • 多态最根本的作用，通过把过程化的条件分支语句转化为对象的多态性，从而消除这些分支语句。
  • 将行为分布在各个对象中，并且这些对象各自负责自己的行为，这正是面向对象设计的优点。

扩展：

JavaScript把函数是一等对象，函数本身也是对象，函数可以用来封装行为并且能够作为参数被四处传递。当我们调用一个函数时，这个函数会返回不同的执行结果，这就是多态表现。所以，高阶函数是一种常见多态实现手段。

封装

封装的目的是将信息隐藏，包括隐藏数据，隐藏实现细节，隐藏设计细节，隐藏对象类型等“任何形式的封装”。封装包括：封装数据，封装实现、封装类型、封装变化。

  • 封装数据（private、public），通过依赖于变量作用域来实现。

出来es6的let，函数可以创建作用域，其中()()主要目的是简化调用，避免全局变量污染。

  • 封装实现
    只需要关心对象暴露出来的接口属性或方法，无需关系其内部是如何实现，内部的修改和调整，只要不需要api，就不会影响到程序其他功能。

  • 封装变化
    在设计模式中，封装被封为：创建型模式、结构型模式、行为型模式。
    核心：找到变化并封装之。

  • 封装类型（动态语言不支持）

原型继承

JavaScript继承不是类继承，而是通过原型委托的方式来实现对象与对象之间的继承。
在原型编程思想中，类并不是必需的，对象未必需要从类中创建而来，一个对象是通过克隆另外一个对象所得到的。
原型模式不仅是一种设计模式，也是一种编程泛型。
原型模式实现关键，是语言本身提供了Object.create（）方法来克隆一个对象。而Object.create（null）可以创建一个没有原型的对象。

原型编程基本规则：

  • 所有的数据都是对象；（基本类型undefined/number/boolean/string/null，对象类型object）
  • 要得到一个对象，不是实例化类，而是找到一个对象作为原型并且克隆它；
    var obj = new Object()或var obj = {},内部引擎自动回从Object.prototype上克隆一个对象

模拟new关键字的实现原理：

  • 对象会记住它的原型；

__proto__是对象与对象构造器原型的纽带

  • 如果对象无法响应某个请求，它会请求委托给自己构造器的原型；
    原型继承的精髓。继承总是发生在对象与对象之间。
    当我们期望一个构造器继承另一个构造器的属性或方法时，可以有选择性的吧对象A的构造器的原型指向到对象B上，从而实现继承。
