##Vue实现响应式原理

vue响应式原理，使用了ECMAScript5中的，对象的属性描述对象Object.defineProperty 里存取器（setter，getter）实现数据对象与DOM对象的双向绑定。参考：阮一峰JavaScript

Vue实现双向绑定流程图

普通 JavaScript 对象传给 Vue 实例的 data 选项，Vue 将遍历此对象所有的属性，并使用 Object.defineProperty把这些属性全部转为 getter/setter。

每个组件实例都有相应的 watcher 实例对象，它会在组件渲染的过程中把属性记录为依赖，之后当依赖项的 setter 被调用时，会通知 watcher 重新计算，从而致使它关联的组件得以更新。

data选项响应式声明属性

优点：data 对象就像组件状态的概要，提前声明所有的响应式属性，可以让组件代码在以后重新阅读或其他开发人员阅读时更易于被理解。

  • 运行的更高效
  • 代码可维护性好
  • 易于被理解

由于 Vue 会在初始化实例时对属性执行 getter/setter 转化过程，所以属性必须在 data 对象上存在才能让 Vue 转换它，这样才能让它是响应的。

data选项异步更新队列

在对Vue中的数据模型data进行数据操作时（增删改查），data数据的改变，不会立即渲染data数据对应的DOM元素，而是，使用原生的 Promise.then 和 MutationObserver或setTimeout(fn, 0)， 异步执行 DOM 更新。即在下一个的事件循环“tick”中，Vue 刷新队列并执行DOM更新工作。

解决方案：
如果你想在 DOM 状态更新后做点什么，请在data数据操作之后立即使用Vue.nextTick(callback)。这样回调函数在 DOM 更新完成后就会调用。
